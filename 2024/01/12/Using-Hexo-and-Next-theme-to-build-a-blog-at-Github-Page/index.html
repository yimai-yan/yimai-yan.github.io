ISSUE 2 : Submodule

Actually this one is kind tricky, after I have solved the problem of wrong url, I found the Github page is still not generated.

Then I checked the Github action, saw the pipeline is failed for:

To solve this issue, here is solution:

  • touch file .gitmodules
  • cd themes/next
  • git remote -v, check the remote origin
  • in the .gitmodules files, add this block
  • git submodule update --init --recursive
  • check configfile in .git folder
  • the submodule block should exists

ISSUE 3 : remote themes/next folder is empty

themes/next remote origin is forked from git@github.com:next-theme/hexo-theme-next.git

The themes/next folder in local is updated with remote origin.

I’ve edited themes/next/_config.yml in local, then in terminal

found nothing changed, the themes/next folder is not tracked by git.

So I assume that’s because themes/next remote origin is git@github.com:next-theme/hexo-theme-next.git, apparently I don’t have access to git push

Change the remote origin to github page repo.

The workflow in github is build success, finally it works.
